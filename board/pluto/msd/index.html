Firmware

ADALM-PLUTO Firmware refers to the U-Boot, HDL, Linux kernel, device drivers, and userspace software, that runs on the Pluto which enales the device to communicate to USB host. This is bundled up and given a specific version number for the ADALM-PLUTO device. For help upgrading firmware, check out the online documentation.

Version Information:

The various parts of the firmware all have their own unique versions as well:

Model #MODEL#
Serial #SERIAL#
Build #BUILD#
Linux #LINUX#
U-Boot #UBOOT#
FPGA #FPGA#
Root FS #ROOTFS#
IIO #IIO#

On the root file system, there are configuration settings that can modify the default configuration, they currently are:

IP Address (PLUTO) #IP#
IP Address (HOST) #HOSTIP#
Netmask #NETMASK#
MAC Address (PLUTO) #MAC#
MAC Address (HOST) #MACHOST#

To change these settings, check the on-line documentation.

Libiio

The Linux Industrial I/O subsystem provides support for devices that in some sense are analog to digital or digital to analog convertors (ADCs, DACs) between people and the Linux kernel. Libiio is a library that has been developed to ease the development of software interfacing to Linux Industrial I/O (IIO) devices. The library abstracts the low-level details of the hardware, and provides a simple yet complete programming interface that can be used for advanced projects. With bindings for custom C, C++, python applications, and applications like MATLAB and Simulink, it can be the cross platform (Windows, MAC, Linux), library that you use to talk to your Pluto device.

IIO Oscilloscope

The IIO Oscilloscope is a example cross platform GTK application, which demonstrates how to interface the ADALM-PLUTO to a GUI. The application supports plotting of the captured data in four different modes (time domain, frequency domain, constellation and cross-correlation), playback of various signals. The application also allows to view and modify settings of the IIO devices on the Pluto.
