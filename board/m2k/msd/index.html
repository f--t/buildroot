Welcome to the ADALM-2000 Active Learning Module

Thank you for purchasing the ADALM-2000 Active Learning Module (M2kSDR). The easy to use ADALM-2000 active learning module (M2kSDR) helps introduce electrical engineering students to the fundamentals of software-defined radio (SDR), radio frequency (RF), and wireless communications. Designed for students at all levels and from all backgrounds, the module can be used for both instructor-led and self-directed learning to help students develop a foundation in real-world RF and communications that they can build on as they pursue science, technology, or engineering degrees.

The M2kSDR works as a portable lab that, when used with a host, can augment classroom learning. MATLAB and Simulink are two of the many software packages supported by M2kSDR, and it provides an intuitive graphical user interface (GUI) so students can learn faster, work smarter, and explore more.

Based on the AD9363, The M2kSDR features independent receive and transmit channels that can be operated in full duplex. The active learning module can generate or acquire RF analog signals from 325 MHz to 3800 MHz at up to 61.44 megasamples per second (MSPS). Small enough to fit in a shirt pocket, the M2kSDR is completely self-contained and entirely USB powered with the default firmware. Because M2kSDR is enabled by libiio drivers, it supports OS X® , Windows®, and Linux®, which allows students to learn and explore on a variety of host platforms.

Features:

  • Separate Receive and Transmit SMA connectors to connect to instrumentation or antennas
    • Software tunable across wide frequency range (325 MHz to 3.8 GHz)
    • Integrated fractional N synthesizers
    • Independent Receive and Transmit local oscillators, with 2.4 Hz step size
  • 12-bit DACs and ADCs
    • channel bandwidth of <200 kHz to 20 MHz
    • Maxium sample rate of 61.44 MSPS (burst mode)
  • Supports time division duplex (TDD) and frequency division duplex (FDD) operation
  • Receive (Rx) gain control
    • Real-time monitor and control signals for manual gain
    • Independent automatic gain control (AGC)
  • USB 2 (480 Mbits/seconds) data and power back to Host Platform
    • Mass storage (for easy firmware updates)
    • Device Firmware Upgrade(for harder firmware upgrades)
    • Serial (for interacting with the Linux kernel/userspace on M2kSDR)
    • Networking/RNDIS (for loading controlling and custom ARM® applications)
    • libiio (bulk USB for SDR data transfer and control)

Package Contents:

  • Analog Devices M2kSDR Active Learning Module
  • Two antennas (824-894 MHz / 1710-2170 MHz)
  • SMA cable
  • USB cable
  • Getting started guide

Libiio

The Linux Industrial I/O subsystem provides support for devices that in some sense are analog to digital or digital to analog convertors (ADCs, DACs) between people and the Linux kernel. Libiio is a library that has been developed to ease the development of software interfacing to Linux Industrial I/O (IIO) devices. The library abstracts the low-level details of the hardware, and provides a simple yet complete programming interface that can be used for advanced projects. With bindings for custom C, C++, python applications, and applications like MATLAB and Simulink, it can be the cross platform (Windows, MAC, Linux), library that you use to talk to your M2kSDR device.

IIO Oscilloscope

The IIO Oscilloscope is a example cross platform GTK application, which demonstrates how to interface the ADALM-2000 to a GUI. The application supports plotting of the captured data in four different modes (time domain, frequency domain, constellation and cross-correlation), playback of various signals. The application also allows to view and modify settings of the IIO devices on the M2kSDR.

GNU Radio

GNU Radio is a free & open-source software development toolkit that provides signal processing blocks to implement software radios. It can be used with readily-available RF hardware, like the M2kSDR to create software-defined radios. It is widely used in hobbyist, and academic environments to support wireless communications research.

GNU Radio performs all the signal processing. You can use it to write applications to receive data out of digital streams or to push data into digital streams, which is then transmitted using hardware.

For more information, check out:
